Section 1

Feature Overview

The Equipment module transforms from a static asset tracker (name, category, status, inspections) into a full financial management system that tracks rates, hours, costs, and revenue per piece of equipment across jobs.

Current State — Asset Tracker
What it does today:
name, year, description, datePurchased, priceOfEquipment, equipmentCategoryId, inspection dates, statusId

Tabs: Overview, Form Submissions, Assigned To, Location, Expenses, Account Payable

Limitations:
No cost/billable rates. No hour tracking. No running hours. No timesheet integration. No job-level equipment P&L.
Target State — Financial System
What it will do (Phases 1-2):
costRatePerHour, billableRatePerHour, runningHours (auto-calculated), rate snapshots

New Tab: Hours Log (with cost/revenue calculations)

Timesheet Integration:
Equipment selector on desktop + mobile timesheets. Meter readings. Billable toggle. Approval flow updates running hours.

Rate Snapshot Concept: When an EquipmentTimeEntry is created, the equipment's current costRatePerHour and billableRatePerHour are captured into costRateSnapshot and billableRateSnapshot. This ensures historical data stays accurate even if rates change later. The calculatedCost and calculatedRevenue are computed at creation time: hours × rateSnapshot.

Meter Meter readings are optional. If provided, end − start is validated against hours (warning if delta doesn't match).

Atomic increment
Dev: Equipment dropdown queries EquipmentAssignment filtered by jobId from the selected timesheet job. Rate snapshots are captured from Equipment.costRatePerHour and Equipment.billableRatePerHour at entry creation time (not at approval). On approval, atomically increment Equipment.runningHours using prisma.equipment.update({ data: { runningHours: { increment: approvedHours } } }). On rejection after prior approval, decrement.

Step 7: Rate Snapshot Logic

On EquipmentTimeEntry creation:

1. Look Up Equipment

Read current rate fields from Equipment record

2. Snapshot Rates

Copy costRatePerHour → costRateSnapshot
Copy billableRatePerHour → billableRateSnapshot

3. Calculate Cost & Revenue

calculatedCost = hours × costRateSnapshot
calculatedRevenue = hours × billableRateSnapshot

4. Null Handling

If rates are null, calculated fields are null

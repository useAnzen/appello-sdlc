C1 — Q21030-9996

Leave Hours Conversion — Michigan Compliance

Michigan state law requires leave to be reported in hours, not days. The system must convert full-day leave into hours using a configurable hours-per-day setting that already exists on the Instance model.

"It's probably going to need to be hours because that's how the state of Michigan's law is read."
— Chris Tremberth, CEO, Rival Insulation
"Yeah, so then eight hours were generated on the CSV."
— Kate Hallett, Office Admin, Rival Insulation

1.1 Leave Report — Before vs After

Reports and CSV exports currently show leave in calendar days. After the change, all leave values display in hours using the instance's fullDayLeaveDuration setting.

C3 — Q21030-10099

Union Subsidy Accrual

Rival Insulation needs automatic accrual of union subsidies based on approved timesheet hours. One flat rate per hour regardless of trade classification. The budgeted amount comes from the estimate; accrued amount is calculated in real-time.

"It needs to accrue the billable subsidy."
— Chris Tremberth, CEO, Rival Insulation (June 2025 onboarding)
"Eventually we would need a form that's turned in so we could submit for the money."
— Chris Tremberth, CEO, Rival Insulation (regarding union submission reports)
What already exists: Subsidy field in estimates (Q21030-5777, In Prod), subsidy invoicing to union halls, manual entry of budgeted subsidy amount. What's missing: Automatic accrual calculation from timesheet hours and a report for union submission.

3A. Leave Hours Conversion — Michigan Compliance (Q21030-9996)

Current State

What already works:

  • Instance.fullDayLeaveDuration (Int, default 28800 seconds = 8 hours) exists in Prisma schema
  • LeaveRequest.ts uses instance?.fullDayLeaveDuration ?? EIGHT_HOURS_AS_SEC when syncing leave to timesheets
  • Partial leave is already tracked in hours

What is hardcoded:

LocationHardcoded ValuePurpose
apps/api/src/models/LeaveRequest.ts line 17 EIGHT_HOURS_AS_SEC = 28800 Fallback constant
apps/frontend/src/lib/utils.ts line ~101 const days = hours / 8 handleHoursToDate() function
apps/frontend/src/pages/reports/employee-leave-report.tsx Calendar day diff Full-day leave counted as days
apps/frontend/src/pages/reports/all-employee-leave.tsx Calendar day diff Same pattern
apps/api/src/graphql/models/employeeLeaveReport.ts Calendar day diff CSV export

Step 1: Fix handleHoursToDate() (utils.ts)

export function handleHoursToDate(totalHours: number, hoursPerDay: number = 8) {
    const days = totalHours / hoursPerDay;
    // ... rest of function
}
All callers pass the instance setting via useSettings() → activeInstance.

Step 2: Update leave reports

From dayjs(endDate).diff(startDate, "day") + 1 (calendar days)
To calendarDays * (fullDayLeaveDuration / 3600) (hours)
Update column headers: "Days" → "Hours"

Step 3: Update CSV export (employeeLeaveReport.ts)

Query the Instance setting and apply the same hours calculation server-side.
